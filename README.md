# AIST Pilot Bot — 14-дневный марафон систематического обучения

**Миссия**: Помочь участникам трансформироваться из случайных учеников в систематических учеников, которые собраны и удерживают внимание на своём развитии.

Telegram-бот для проведения 14-дневного марафона от [Мастерской инженеров-менеджеров](https://system-school.ru/).

## Формат марафона

- **14 дней** — по 2 темы каждый день (теория + практика)
- **Теория** — персонализированный текст с вопросом для размышления
- **Практика** — задание с рабочим продуктом (текст, список, схема)
- **Персонализация** — контент адаптируется под профиль участника
- **Гибкость** — можно нагнать до 1 пропущенного дня (макс 4 темы/день)
- **Напоминания** — автоматическая отправка + 2 повторных напоминания

## Целевая аудитория

Три состояния застревания:
- **Хаос** — внимание разбросано, много незавершённых дел
- **Тупик** — стабильность без роста, "день сурка"
- **Поворот** — хочется перемен, но страшно менять

## Быстрый запуск

Найдите бота в Telegram: [@aist_pilot_bot](https://t.me/aist_pilot_bot)

Напишите `/start` и пройдите короткий онбординг (2 минуты):
1. Представьтесь — имя и чем занимаетесь
2. Выберите интересы из списка
3. Ответьте на 2 вопроса о ваших целях
4. Выберите время на тему (5-25 минут)
5. Укажите удобное время для напоминаний (по Москве)
6. Выберите дату старта марафона

После онбординга бот будет присылать материалы каждый день в выбранное время.

## Как пользоваться ботом

### Команды

| Команда | Описание |
|---------|----------|
| `/start` | Начать марафон (онбординг + выбор даты старта) |
| `/learn` | Получить следующую тему |
| `/progress` | Посмотреть прогресс по дням |
| `/profile` | Показать профиль |
| `/update` | Обновить настройки профиля |
| `/help` | Справка по командам |

### Процесс марафона

```
1. /start → Онбординг за 2 мин + выбор даты старта марафона
2. Каждый день бот присылает 2 темы (теория + практика)
3. Теория: читаешь материал (5-25 мин) + отвечаешь на вопрос
4. Практика: выполняешь задание + отправляешь название рабочего продукта
5. День засчитан
6. Пропустил? Можно нагнать 1 день (макс 4 темы за день)
```

### Правила прохождения

| Правило | Значение |
|---------|----------|
| Тем в день (норма) | 2 (теория + практика) |
| Тем в день (максимум) | 4 (нагнать 1 день) |
| Нагнать за раз | Только 1 пропущенный день |
| Идти вперёд | Нельзя раньше дня марафона |

### Напоминания

- **В указанное время** — отправка контента дня
- **+1 час** — первое напоминание если не начал
- **+3 часа** — последнее напоминание

### Настройки профиля (/update)

- **Имя, занятие, интересы** — для персонализации примеров
- **Что важно в жизни** — для мотивационных блоков
- **Что хочу изменить** — для связи материала с целями
- **Время на тему** — 5, 10, 15, 20 или 25 минут
- **Расписание** — время ежедневного напоминания (UTC+3 Москва)

## Как создать такого бота

Инструкция для разработчиков, которые хотят развернуть собственную копию бота.

### Шаг 1: Создать Telegram-бота (2 мин)

1. Откройте Telegram и найдите [@BotFather](https://t.me/BotFather)
2. Отправьте `/newbot`
3. Введите имя бота: `AIST Pilot`
4. Введите username: `aist_pilot_bot` (придумайте уникальный)
5. **Скопируйте токен** — он выглядит так: `7123456789:AAHxxxxxxxxxxxxxxxxxxxxx`

### Шаг 2: Получить Anthropic API ключ (5 мин)

1. Зайдите на [console.anthropic.com](https://console.anthropic.com)
2. Зарегистрируйтесь или войдите
3. Перейдите в **Settings → API Keys**
4. Нажмите **Create Key**
5. **Скопируйте ключ** — он выглядит так: `sk-ant-api03-xxxxx...`

> Стоимость: ~$3-5 на 1000 тем

### Шаг 3: Запустить на Railway (8 мин)

#### 3.1 Форкните репозиторий

1. Нажмите кнопку **Fork** вверху этой страницы
2. Выберите свой аккаунт GitHub
3. Нажмите **Create fork**

#### 3.2 Разверните на Railway

1. Зайдите на [railway.app](https://railway.app)
2. Нажмите **Login** → войдите через GitHub
3. Нажмите **New Project**
4. Выберите **Deploy from GitHub repo**
5. Найдите и выберите `aist_pilot_bot`
6. Railway начнёт деплой (пока с ошибкой — это нормально)

#### 3.3 Добавьте переменные окружения

1. В Railway нажмите на ваш сервис
2. Перейдите во вкладку **Variables**
3. Добавьте переменные (нажимайте **+ New Variable**):

| Имя | Значение |
|-----|----------|
| `TELEGRAM_BOT_TOKEN` | Токен из Шага 1 |
| `ANTHROPIC_API_KEY` | Ключ из Шага 2 |

4. Нажмите **Deploy** (или подождите автодеплой)

#### 3.4 Готово!

Через 1-2 минуты бот заработает. Найдите его в Telegram и напишите `/start`

### Конфигурация

#### Переменные окружения

| Переменная | Обязательно | Описание |
|------------|-------------|----------|
| `TELEGRAM_BOT_TOKEN` | Да | Токен от @BotFather |
| `ANTHROPIC_API_KEY` | Да | Ключ API Anthropic |
| `DATABASE_URL` | Да | URL PostgreSQL базы данных |
| `MCP_URL` | Нет | URL MCP руководств (по умолчанию: guides-mcp.aisystant.workers.dev) |

#### Структура знаний

Темы для изучения задаются в файле `knowledge_structure.yaml`:

```yaml
sections:
  - id: "section-1"
    title: "Название раздела"
    topics:
      - id: "1-1-topic-id"
        title: "Название темы"
        order: 1
        main_concept: "ключевое понятие"
        related_concepts:
          - "понятие1"
          - "понятие2"
        key_insight: "Главный инсайт темы"
        pain_point: "Какую боль решает тема"
        source: "Источник материала"
```

#### MCP интеграция

Бот автоматически обогащает контент из руководств Aisystant через MCP (Model Context Protocol):
- Semantic search по теме находит релевантные фрагменты
- Контекст передаётся в Claude для генерации персонализированного материала
