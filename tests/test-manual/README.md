# Ручное тестирование AIST Track Bot

Эта папка содержит материалы для ручного тестирования бота.

## Содержимое

| Файл | Описание |
|------|----------|
| `testing-scenarios.md` | Полный чек-лист сценариев тестирования (123 сценария) |
| `create-testing-spreadsheet.js` | Google Apps Script для создания таблицы тестирования |

---

## Как развернуть таблицу тестирования в Google Sheets

### Шаг 1: Создайте новую таблицу

1. Откройте [Google Sheets](https://sheets.google.com)
2. Нажмите **+ Пустой файл** (или **Blank spreadsheet**)
3. Назовите таблицу: `AIST Track Bot - Тестирование`

### Шаг 2: Откройте редактор скриптов

1. В меню выберите **Расширения** → **Apps Script**
   - (Или **Extensions** → **Apps Script** на английском)
2. Откроется новая вкладка с редактором скриптов

### Шаг 3: Вставьте код скрипта

1. В редакторе скриптов удалите весь код по умолчанию (обычно там `function myFunction() {}`)
2. Откройте файл `create-testing-spreadsheet.js` из этой папки
3. Скопируйте **весь** код из файла
4. Вставьте в редактор скриптов
5. Нажмите **Ctrl+S** (или **Cmd+S** на Mac) для сохранения
6. Назовите проект: `AIST Testing Setup`

### Шаг 4: Запустите скрипт

1. В редакторе скриптов:
   - Найдите выпадающий список функций (рядом с кнопкой "Выполнить")
   - Выберите функцию `createTestingSpreadsheet`
2. Нажмите кнопку **▶ Выполнить** (Run)

### Шаг 5: Авторизуйте скрипт

При первом запуске Google попросит разрешения:

1. Нажмите **Просмотреть разрешения** (Review permissions)
2. Выберите ваш Google аккаунт
3. Появится предупреждение "Google не подтвердил это приложение":
   - Нажмите **Дополнительно** (Advanced)
   - Нажмите **Перейти к AIST Testing Setup (небезопасно)** (Go to AIST Testing Setup (unsafe))
4. Нажмите **Разрешить** (Allow)

### Шаг 6: Дождитесь создания таблицы

- Скрипт создаст все листы и заполнит их данными
- Процесс занимает около 30-60 секунд
- В журнале выполнения (внизу) появится сообщение о завершении

### Шаг 7: Проверьте результат

Вернитесь на вкладку с Google Sheets и обновите страницу (**F5**).

Должны появиться листы:
- **Сводка** — общая статистика тестирования
- **0. Предложения** — секция для замечаний по контенту
- **1. Регистрация** — сценарии 1.1-1.12
- **2. Марафон** — сценарии 2.1-2.17
- **3. Лента** — сценарии 3.1-3.14
- ... и так далее до раздела 11
- **Баги** — лист для отчётов о багах

## Как использовать таблицу

### Заполнение результатов

1. Перейдите на нужный лист (например, **2. Марафон**)
2. В колонке **Статус** выберите из выпадающего списка:
   - `✅ OK` — сценарий пройден
   - `⚠️ Замечание` — работает, но есть UX-проблемы
   - `❌ Баг` — не работает или работает неправильно
   - `⏭️ Пропущен` — не тестировался
3. В колонке **Комментарий** добавьте заметки

### Раздел "Предложения" (Секция 0)

Используйте лист **0. Предложения** для:
- Замечаний по контенту (тексты уроков, дайджестов)
- Предложений по UX улучшениям
- Идей по новым функциям

### Отслеживание багов

1. Перейдите на лист **Баги**
2. Заполните информацию о баге:
   - ID бага (BUG-001, BUG-002...)
   - Связанный сценарий
   - Описание
   - Серьёзность
   - Статус

### Просмотр сводки

Лист **Сводка** автоматически подсчитывает:
- Общее количество сценариев по статусам
- Прогресс тестирования по разделам
- Количество открытых багов

---

## Примечания

- Скрипт можно запустить повторно — он создаст новые листы (старые останутся)
- При необходимости обновить структуру — удалите листы и запустите скрипт заново
- Таблица настроена для совместной работы — можно дать доступ другим тестировщикам
